Welcome to today's lab session! In this lab, we will explore the concepts of statelessness in the HTTP protocol and different approaches to authentication.

The HTTP protocol is stateless, meaning that each request made to the server is independent and doesn't have any connection or relationship with previous requests. However, to maintain continuous authentication and avoid repeatedly asking users to log in, we can utilize session-based or token-based authentication methods.

In this lab, we will focus on the token-based authentication approach. Token-based authentication involves using tokens, typically in the form of JSON Web Tokens (JWT), to authenticate and authorize requests. This approach allows for efficient and secure authentication without relying on server-side sessions.

2. Handling Login Requests

In the index.js file of your Express application, implement a new route to handle login requests:

router.post('/login', async function (req, res, next) {
  const db = await connectToDB();
  try {
    // check if the user exists
    var user = await db.collection("users").findOne({ email: req.body.email });
    if (!user) {
      res.status(401).json({ message: 'User not found' });
      return;
    }

    res.json(user);
    
  } catch (err) {
    res.status(400).json({ message: err.message });
  } finally {
    await db.client.close();
  }
});

In this route, we first check if the requested user exists in the system. If the user is found, we send back the user object as a response. Please note that for simplicity, we assume that any user found in the system can log in successfully. Additionally, if the user object contains a password field, it should be deleted before sending the response.

4. Generating JWT Bearer Token Authentication

To generate JWT Bearer Token, follow the steps below:

In the app.js file of your backend application, add the following line:

process.env.TOKEN_SECRET = 'secret';

Please note that using a string for process.env.TOKEN_SECRET in app.js is not secure, but it's acceptable for this lab exercise. In a real-world scenario, you should use a more secure approach, such as environment variables, which can be discussed during deployment.

Next, install the jsonwebtoken dependency by running the following command:

npm install jsonwebtoken --save

This package allows you to generate and verify JSON Web Tokens.

Create a new file named auth.js in the utils folder and add the following code:

const jwt = require('jsonwebtoken');

// generate a token
const generateToken = function (user) {
    return jwt.sign(user, process.env.TOKEN_SECRET, {
        expiresIn: 86400 // expires in 24 hours
    });
}

module.exports = { generateToken };

In this code, the backend server signs a token with a secret (process.env.TOKEN_SECRET) and sets the token's expiry time to 24 hours. The token payload contains the user object.

In the index.js file, import the generateToken function by adding the following line:

const { generateToken } = require('../utils/auth');

This allows you to use the generateToken function to create a JWT token.

Then, update the /login route handler with the following

// res.json(user);

delete user.password;
delete user.ip_address;

// generate a JWT token
const token = generateToken(user);

// return the token
res.json({ token: token });

In this code, the password and IP address fields are removed from the user object. Then, a JWT token is generated using the generateToken function, and the token is sent back to the client in JSON format.

5. Authorization with Passport.js

This section explains how to implement authorization using Passport.js and JWT Bearer Tokens. By utilizing Passport.js, a popular authentication middleware, and JSON Web Tokens (JWT) as the authentication mechanism, you can secure your routes and ensure that only authenticated users with valid tokens can access protected resources.

Install the required modules by running the following command:

npm install passport passport-http-bearer --save

This installs the Passport.js module for authentication and the BearerStrategy module for handling Bearer token authentication.

In the app.js file, add the necessary code around line 10.

var jwt = require('jsonwebtoken');
var passport = require('passport');
var BearerStrategy = require('passport-http-bearer').Strategy;
passport.use(new BearerStrategy(
  function (token, done) {
    jwt.verify(token, process.env.TOKEN_SECRET, function (err, decoded) {
      if (err) { return done(err); }
      return done(null, decoded, { scope: "all" });
    });
  }
));

This code configures Passport.js to use the BearerStrategy for token authentication. It verifies the received token using the same secret that was used to sign it.

Protect the routes under /users by replacing the existing code:

app.use('/users', usersRouter);

with:

app.use('/users', passport.authenticate('bearer', { session: false }), usersRouter);

This code applies the passport.authenticate('bearer', { session: false }) middleware to the /users routes, ensuring that only authenticated requests with a valid Bearer token can access the protected routes.

Now, open the Thunder Client and make a request to http://localhost:3000/users. Since the route is protected with token authentication, an Unauthorized response will be returned.

6. Managing Bookings by Users

To manage bookings by a user (staff member), the following new route is added in bookings.js:

var passport = require('passport');

// Specify booking being managed by a user
router.patch('/:id/manage', passport.authenticate('bearer', { session: false }), async function (req, res) {
    const db = await connectToDB();
    try {
        let result = await db.collection("bookings").updateOne({ _id: new ObjectId(req.params.id) },
            {
                $set: { manager: new ObjectId(req.user._id) }
            });

        if (result.modifiedCount > 0) {
            res.status(200).json({ message: "Booking updated" });
        } else {
            res.status(404).json({ message: "Booking not found" });
        }
    } catch (err) {
        res.status(400).json({ message: err.message });
    }
    finally {
        await db.client.close();
    }
});

In this route, the PATCH method is used to update the booking document with the manager's ID. The route is protected using the passport.authenticate middleware with Bearer token authentication. If the update is successful, a 200 response with the message "Booking updated" is sent. If the booking is not found, a 404 response is sent. Any other errors are handled with a 400 response.

To test this route, you can use Thunder Client and make a PATCH request to the endpoint http://localhost:3000/bookings/650bca93fc13ae7058fd5414/manage (replace 650bca93fc13ae7058fd5414 with the actual booking ID you want to manage). This will assign the booking to the authenticated user as the manager.

7. Retrieving Users and Their Bookings

To retrieve all users and their bookings, provide the /with/bookings route in users.js as follows:

const { connectToDB, ObjectId } = require('../utils/db');

router.get('/with/bookings', async function (req, res) {
  const db = await connectToDB();
  try {
    let result = await db.collection("users").aggregate([
      {
        $lookup: {
          from: "bookings",
          localField: "_id",
          foreignField: "manager",
          as: "bookings"
        }
      },
      // remove the ip_address field
      { $project: { ip_address: 0 } }
    ]).toArray();
    res.json(result);
  } catch (err) {
    res.status(400).json({ message: err.message });
  }
  finally {
    await db.client.close();
  }
});

In this route, the $lookup stage is used to join the users collection with the bookings collection based on the manager field. The resulting documents include the corresponding bookings, which are stored in the bookings property. The ip_address field is excluded from the result using the $project stage.

<script setup>

In the provided code, response.ok is used to check whether the HTTP response from the server indicates success or failure.

The response.ok property is a Boolean value that is true if the response status is within the range 200-299 (indicating a successful request), and false otherwise. It provides a convenient way to determine if the request was successful without manually checking the status code.

In the login functionality, response.ok is used to validate the response from the server after the login request is made. If response.ok is true, it means the login request was successful and the server has responded with a status code indicating success (e.g., 200 OK). In this case, the returned token is stored in the browser's local storage.

On the other hand, if response.ok is false, it means the server has responded with an error status code (e.g., 400 Bad Request, 401 Unauthorized). In this case, an error message is thrown, and an alert is displayed to the user with the error message.

9. Using the Token

To access user routes that require authentication, you need to include the token in the request headers. The following code demonstrates how to implement this functionality in the UsersView.vue file:

<script setup>
import { onMounted, ref } from "vue";

const users = ref([]);
const name = ref("");

const loadAsyncData = async () => {
    try {
        // Get the token from local storage    
        const token = localStorage.getItem('token');

        // Send a request to the endpoint with the token in the Authorization header
        var response = await fetch("/api/users/with/bookings", {
            headers: {
                Authorization: `Bearer ${token}`,
            }
        });

        if (!response.ok) {
            throw new Error(response.statusText);
        }

        // convert the response to json
        const json = await response.json();
        // log the json
        console.log(json);
        // set the data
        users.value = json;
    } catch (error) {
        console.log(error);
    }
};

onMounted(() => {
    loadAsyncData();
});
</script>

<template>
    {{ name }}
    <div v-for="user in users" :key="user._id">
        {{ user }}
    </div>
</template>

In this code, the loadAsyncData function is responsible for fetching the user data from the server. It first retrieves the token from the browser's local storage using localStorage.getItem('token').

Next, a request is made to the /users/with/bookings endpoint. The token is added to the request headers using the Authorization field, with the value being Bearer ${token}. This follows the standard convention for including tokens in the HTTP Authorization header.

If the response status is not within the successful range (200-299), an error is thrown with the response status text.

Upon receiving a successful response, the JSON data is extracted using response.json(). The retrieved data is then assigned to the users ref, making it reactive and triggering a re-render of the template to display the user data.

11. Logout

To implement the logout functionality, you need to remove the token from the local storage. In your UsersView.vue component, you can define a logout method as follows:.

const logout = function() {
  localStorage.removeItem('token');
  location.reload()
}

The logout method removes the token from the local storage using localStorage.removeItem('token'). Additionally, it reloads the page using location.reload() to reset the application state and redirect the user to a non-authenticated state.

To trigger the logout method, you can add a button in your template with the @click event handler:

<button type="button" class="btn btn-primary my-4" @click="logout">Log Out</button>

When the "Log Out" button is clicked, the logout method will be called, removing the token from the local storage and reloading the page, effectively logging the user out of the application.

12. Date and Time

To work with dates and times in JavaScript, you can use the Date object. It provides various methods to retrieve and display date and time values in different formats.

To create a new Date object, you can use the following syntax:

> foo = new Date()
2023-11-04T09:19:38.766Z

> foo = new Date("2011-05-03")
2011-05-03T00:00:00.000Z

To retrieve and display the date and time in a more readable format, you can use various methods available on the Date object.

For example, the toString() method returns a string representation of the Date object:

foo.toString();
// Output: 'Tue May 03 2011 08:00:00 GMT+0800 (Hong Kong Standard Time)'

The toDateString() method returns a string representing the date part of the Date object:

foo.toDateString();
// Output: 'Tue May 03 2011'

If you want to remove the day of the week from the date string, you can use the slice() method:

foo.toDateString().slice(4);
// Output: 'May 03 2011'

The toLocaleDateString() method allows you to specify a locale and returns a string representing the date part in a localized format. For example, in the en-CA (English-Canada) locale:

foo.toLocaleDateString("en-CA");
// Output: '2011-05-03'

12.1. Time Difference

To calculate the time difference between two dates, you can subtract one Date object from another. The result will be the difference in milliseconds.

> new Date() - new Date("2011-05-03")
437817818019

To obtain the number of days between the dates, you can divide the difference by the number of milliseconds in a day (24 hours * 60 minutes * 60 seconds * 1000 milliseconds). Alternatively, you can use third-party libraries like date-fns to simplify date calculations.

To use the differenceInDays function from the date-fns library, you need to install it first:

Then, in your BookingView.vue component, you can import the differenceInDays function:
