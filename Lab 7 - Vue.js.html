The Application Instance

Let's clone index.html and name it create.html. In the newly created create.html file, please ensure that the attribute id="app" is added to the <body> DOM element. Next, develop the following <script> element:

<script type="module">
	import { createApp } from 'https://unpkg.com/vue@3/dist/vue.esm-browser.js'

	createApp({
		data() {
			return {
				message: 'Hello Vue!'
			}
		}
	}).mount('#app')
</script>

To start a Vue application, we need to create a new application instance using the createApp() function.

The createApp() function is called with an options object that defines the configuration and behavior of the Vue application. This options object typically includes properties like data, methods, computed, components, and more.

The createApp() function returns an application instance that represents the Vue application. This instance allows you to interact with and control the application.

Once you have the application instance, you can use the .mount() method to specify the HTML element where the Vue application will be mounted/rendered. This is typically done by passing a CSS selector that targets the desired element in the DOM.

The Data Properties

In the Options API, the data option is used to define the initial data properties of a Vue application/component. The data option should be a function that returns an object containing the desired data properties and their initial values.

To develop data property under data(), you can do the following:

// message: 'Hello Vue!'
booking: {
	email: '',
	numTickets: 1,
	payment: 'Credit Card',
	team: '',
	superhero: '',
	terms: false
}

In the above example, the booking property is added to the application's data function. It contains several properties, each initialized with their respective default values.

These data properties will be reactive, meaning that any changes to them will trigger re-rendering of the application and automatically update any bindings or references to these properties in the template or other parts of the application.

Declarative Programming and Directives

Vue.js is built on the belief that declarative programming should be used to create user interfaces and connect software components.

To support this approach, Vue.js provides a set of directives that enable declarative programming. Some of the commonly used directives include v-model and v-on.

Using v-model Directive for Two-Way Data Binding in Forms

The v-model directive is commonly used to create two-way data bindings on form input and select elements in Vue.js. To modify the form elements, we can replace all name attributes with v-model. For instance, name="email" should be replaced with v-model="booking.email".

By using v-model on a checkbox, the booking.terms property will be bound as a boolean value instead of a string. This is because checkboxes represent boolean values (true or false).

Using v-on Directive for Event Handling

We can use the v-on directive in Vue.js to listen to DOM events and execute JavaScript code when those events are triggered.

To apply the v-on directive to the form, you can use the following syntax:

<form class="container my-5" v-on:submit.prevent="submitBooking">

The .prevent modifier is used in this example to prevent the default form submission behavior.

Vue.js provides a shorthand for the v-on directive. For example, v-on:submit can be shortened to @submit. Here's an updated version of the form using the shorthand notation:

<form class="container my-5" @submit.prevent="submitBooking">

The Methods Option

Develop the following methods option below data():

methods: {
    // post the booking to the backend
    async submitBooking() {
        // post the booking to the backend
        const response = await fetch('/bookings', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json'
            },
            body: JSON.stringify(this.booking)
        });
        // convert the response to json
        const json = await response.json();
        // log the json
        console.log(json);
        // alert the user
        alert(JSON.stringify(json));
    }
}

In this code, we have implemented a submitBooking method to initiate an AJAX call.

Inside the methods option, the data properties are accessible using this.

The submitBooking method uses the fetch function to send a POST request to the /bookings endpoint. The request includes the headers specifying the content type as JSON, and the body is populated with the stringified version of this.booking.

Single-page Application (SPA)

Vue.js is a progressive JavaScript framework for building user interfaces and single-page applications (SPAs). It allows developers to build reactive UIs that efficiently update and render only components that need to change.

A Vue.js SPA dynamically updates the content on a single page without requiring full page reloads. When the user interacts with the app, Vue.js handles updating the necessary components and content on the fly rather than refreshing the entire page. This creates faster, smoother experiences compared to traditional multi-page apps.

To create a new Vue.js project, we can use create-vue, a scaffolding tool that sets up a development environment with Vue.js installed and configured.

Open the terminal and run the following command:

npx create-vue lab-spa

When prompted, choose No for all options except Vue Router (#3) and ESLint (#7). Adding Vue Router enables client-side routing in the app for navigating between different components. ESLint is a static analysis tool commonly used in JavaScript development to enforce code quality and maintain consistent coding practices.

Once the project is created, navigate into the directory, install the dependencies and start the development server:

cd lab-spa
npm install
npm run dev

This will launch the Vue.js app on port 5173. You can now visit http://localhost:5173 in your browser to view the running Vue.js single page application!

Project Structure

Here are some key points about the project structure created by create-vue for a Vue.js app:

  • src/ - Contains the source code for the Vue app. This is where components, views, and assets go.
  • src/assets/ - Asset files like images, fonts, etc.
  • src/components/ - Vue components (.vue files) which make up the reusable building blocks of the app.
  • src/views/ - The different views or pages that make up the app. Built out of components.
  • src/router/ - The Vue Router configuration for routing between views/pages.
  • src/App.vue - The root component that contains the router view.
  • src/main.js - The entry point that boots up and configures the Vue app.
  • package.json - Configures dependencies, scripts like dev, build etc.
  • vite.config.js - Build configuration for Vite, the build tool.
  • index.html - The root HTML file that is served.

<script setup>
import { ref, onMounted, computed, watch } from 'vue'

const booking = ref({
    email: '',
    numTickets: 1,
    payment: 'Credit Card',
    team: '',
    superhero: '',
    terms: false
})

const submitBooking = async function () {
    // post the booking to the backend
    const response = await fetch('/api/bookings', {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json'
        },
        body: JSON.stringify(booking.value)
    });
    // convert the response to json
    const json = await response.json();
    // log the json
    console.log(json);
    // alert the user
    alert(JSON.stringify(json));
}
</script>

In this code snippet, we are utilizing the Composition API. The ref() function is used to create a reactive and mutable booking object. The booking object holds properties, which are initialized with their default values.

The use of booking.value allows us to access the inner value of the booking ref object, similar to how we access instance properties in the Options API.

Computed Property

In Vue.js, computed properties allow us to declare derived data that depends on other state. The computed property will automatically re-compute and update whenever its dependent data changes.

We can create a computed property using the computed() method:

// Use computed property to get the superheroes
const superheroes = computed(() => {
    if (booking.value.team == 'Avengers') {
        return ["Captain America", "Iron Man", "Thor", "Hulk", "Black Widow", "Hawkeye"];
    } else if (booking.value.team == "JLA") {
        return ["Superman", "Batman", "Wonder Woman", "Flash", "Green Lantern", "Aquaman"];
    } else {
        return [];
    }
});

The computed method takes a getter function that returns the computed value. This will be called automatically when dependencies change.

Note that booking is a ref object, so we access the inner value via booking.value.

As a result, the onchange attribute could also be removed.

List Rendering with v-for

The code below uses the v-for directive to render a list of superheroes as <option> elements in the second dropdown menu:

<option v-for="superhero in superheroes" :key="superhero" :value="superhero"> {{ superhero }} </option>

In this code, the v-for directive is used to iterate over the superheroes array. For each superhero in the array, a corresponding <option> element is rendered.

:key="superhero": This is a special attribute used to provide a unique identifier for each rendered element in a list. It helps Vue.js efficiently update the DOM when the list changes. In this case, the superhero variable is used as the unique key for each <option> element.

Mustache

In Vue.js, Mustache refers to the double curly braces ({{ }}) syntax used for data interpolation and rendering in templates. It is inspired by the Mustache template language.

Within the Mustache syntax, you can place an expression or variable name inside the curly braces, and Vue.js will evaluate that expression and replace it with the corresponding value. This allows you to dynamically display data in your Vue templates.

In the example provided, the name of each superhero will be displayed within the <option> element.

Watcher

To pre-select the first element from the second drop-down list whenever the user makes a change to the first one, you can implement the watcher functionality for the team value. Follow these steps:

  1. Create a watcher using the watch() function to monitor changes in the booking.value.team property.
  2. In the watcher's callback function, set the booking.value.superhero property to the first element of the superheroes.value array.

Here's an example implementation:

watch(() => booking.value.team, (newValue) => {
    // if the new value is not empty
    if (newValue) {
        booking.value.superhero = superheroes.value[0];
    } else {
        booking.value.superhero = "";
    }
});

Proxy Server

In order to enable the Vue app to consume the Express endpoints, we need to set up a proxy configuration that will route the requests to the appropriate endpoints.

To achieve this, open the vite.config.js file and add the following code inside the server object of the defineConfig() function:

server: {
	proxy: {
	  '/api': {
			target: 'http://localhost:3000',
			changeOrigin: true,
			rewrite: (path) => path.replace(/^\/api/, ''),
	  }
	}
},

With this configuration, any web request starting with /api will be automatically redirected to our Express server running on port 3000. The changeOrigin property is set to true, which ensures that the Origin header of the request is modified to match the target server's origin.

The rewrite arrow function that takes a path parameter and applies a replacement operation using a regular expression. It uses the replace() method to replace the part of the path string that matches the regular expression /^\/api/ with an empty string.

In other words, it removes the prefix /api from the path string if it exists at the beginning of the string. The resulting modified path is returned as the output of the arrow function.

Now, you can test your work by submitting data through the form. The requests made to /api will be seamlessly routed to the Express backend, allowing your Vue app to interact with the backend endpoints effectively.

Viewing a Single Booking

To display a single booking in a Vue SPA app, we need to set up a new route in the /router/index.js file. Open the file and add the following code:

{
	path: '/booking/:id',
	name: 'view-booking',
	// route level code-splitting
	// this generates a separate chunk (about.[hash].js) for this route
	// which is lazy-loaded when the route is visited.
	component: () => import('../views/BookingView.vue')
},

In this code snippet, we define a dynamic path parameter :id that represents the unique identifier of the booking. This allows us to access a specific booking by its ID. The component property points to the BookingView.vue file, which will be loaded lazily when the route is visited.

Now, let's navigate to the BookingView.vue file. To obtain the dynamic path parameter (id), we need to import the useRoute function from vue-router in the script section of the BookingView.vue file:

import { useRoute } from "vue-router";
const route = useRoute();

To retrieve a booking from the backend, we can utilize the route object obtained from useRoute(), where the path parameters are accessible under route.params. To accomplish this, let's develop the following function in the BookingView.vue file

// a function to get the booking from the backend
const getBooking = async function () {
    // get the booking from the backend
    const response = await fetch('/api/bookings/' + route.params.id);
    // convert the response to json
    const json = await response.json();
    // log the json
    console.log(json);
    // set the booking
    booking.value = json;
}

The onMounted Lifecycle Hook

During the creation of a Vue instance, there are various initialization steps and lifecycle hooks that can be utilized to execute custom code at specific stages.

The onMounted hook is particularly useful as it is triggered once the component has been mounted, meaning the DOM elements are ready. We can take advantage of this hook to perform actions after the component has been rendered.

In your BookingView.vue component, locate our watcher and move it inside the onMounted hook as follows:

onMounted(async () => {
    // if there is an id in the route
    if (route.params.id) {
        await getBooking();
    }

    watch(() => booking.value.team, (newValue) => {
        // if the new value is not empty
        if (newValue) {
            booking.value.superhero = superheroes.value[0];
        } else {
            booking.value.superhero = "";
        }
    });
});

Within the onMounted hook, we check if the id parameter is available in the route. If it exists, we call the getBooking function to fetch and populate the booking data. This ensures that the getBooking function is only invoked when an id is present in the route, indicating that it's a request to view an existing booking. Otherwise, if no id is provided, it suggests that it's a request to create a new booking.

The onMounted function registers a callback function to be called after the component has been mounted.

We enable the watcher only after the getBooking() function is completed to ensure that the retrieved superhero field is not overwritten by the first element of the superheroes array.

Conditional Rendering with v-if

To conditionally render a read-only section, you can use the v-if directive. The v-if directive allows you to conditionally render elements based on the truthiness of an expression. In this case, the expression route.name == 'view-booking' is used to check if we are not in the viewing mode.

Here's an example of how a new section can be conditionally rendered:

With this code, the form will only be rendered when we are not in the view mode.
